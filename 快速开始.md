# 快速开始

## 环境准备

### 1. 安装 ESP-IDF v6.0.1

```bash
# 安装依赖
sudo apt-get install git wget flex bison gperf python3 python3-pip python3-venv \
    cmake ninja-build ccache libffi-dev libssl-dev dfu-util libusb-1.0-0

# 克隆 ESP-IDF
mkdir -p ~/esp
cd ~/esp
git clone -b v6.0.1 --recursive https://github.com/espressif/esp-idf.git

# 安装工具链
cd esp-idf
./install.sh esp32c3

# 设置环境变量
echo "source ~/esp/v6.0.1/esp-idf/export.sh" >> ~/.bashrc
source ~/.bashrc
```

### 2. 验证安装

```bash
idf.py --version
# 应输出: ESP-IDF v6.0.1
```

## 编译固件

### 方法一：使用 build.sh 脚本

```bash
cd esp32c3
bash build.sh
```

脚本自动执行：
1. 生成版本时间戳头文件
2. 清理旧编译
3. 配置目标芯片 (ESP32-C3)
4. 编译固件
5. 拷贝到 output/ 目录

编译完成后，`output/` 目录包含：
- `app.bin` — 主固件 (~1.2MB)
- `bootloader.bin` — 引导程序 (~21KB)
- `partition-table.bin` — 分区表 (~3KB)
- `ota_data_initial.bin` — OTA 数据 (~8KB)

### 方法二：手动编译

```bash
cd esp32c3
source ~/esp/v6.0.1/esp-idf/export.sh
idf.py set-target esp32c3
idf.py build
```

## 烧录固件

### 方法一：esptool 命令行

```bash
# 查看串口
ls /dev/ttyUSB*

# 烧录（替换为实际串口）
esptool.py --chip esp32c3 \
    --port /dev/ttyUSB0 \
    --baud 460800 \
    write-flash \
    0x0 output/bootloader.bin \
    0x8000 output/partition-table.bin \
    0x11000 output/ota_data_initial.bin \
    0x20000 output/app.bin
```

### 方法二：Flash Download Tool (Windows)

1. 下载 [Flash Download Tool](https://www.espressif.com/en/support/download/other-tools)
2. 选择芯片类型：ESP32-C3
3. 配置固件文件：

| 文件 | 地址 |
|------|------|
| bootloader.bin | 0x0 |
| partition-table.bin | 0x8000 |
| ota_data_initial.bin | 0x11000 |
| app.bin | 0x20000 |

4. 设置参数：
   - SPI Speed: 80MHz
   - SPI Mode: DIO
   - Flash Size: 4MB
5. 选择串口，点击 START

### 方法三：idf.py

```bash
idf.py -p /dev/ttyUSB0 flash
```

## 基本使用

### 独立采集模式（默认）

上电后自动进入独立采集模式：
1. 关闭 BLE 和 WiFi
2. CPU 降至最低频率 (10MHz)
3. MAX30102 开始采集
4. 数据保存到 TF 卡
5. 1 分钟无数据自动进入 Deep-sleep

### BLE 配对模式

1. **长按 BUTTON1 (GPIO11) 3 秒**
2. 系统进入 BLE 广播模式
3. 使用手机 App 搜索并连接 "PPG-Monitor"
4. 连接成功后可通过 App 控制设备

### WiFi 模式

1. **双击 BUTTON1 (GPIO11)**（间隔 < 500ms）
2. 系统自动连接已保存的 WiFi
3. 通过浏览器访问设备 IP 进行文件管理
4. 3 分钟无活动自动关闭

### 切换工作模式

1. **单击 BUTTON1 (GPIO11)**
2. 切换 Standalone ↔ Measuring 模式

## TF 卡使用

### 格式化要求

- 文件系统：FAT32
- 容量：建议 4GB-32GB
- 速度：Class 10 或更高

### 存储结构

```
/sdcard/
├── raw/                    # 原始 PPG 数据
│   └── 20260617_120000.bin.lz4
├── csv/                    # 心率/血氧结果
│   └── 20260617.csv
└── log/                    # 系统日志
    └── 20260617.log
```

### CSV 文件格式

```csv
timestamp,hr,spo2,hr_valid,spo2_valid
2026-06-17 12:00:00,72,98,1,1
2026-06-17 12:00:05,75,97,1,1
2026-06-17 12:00:10,73,98,1,1
```

## OTA 升级

### Web OTA

1. 连接设备 WiFi 或确保设备在同一局域网
2. 浏览器访问 `http://设备IP/api/ota`
3. 选择固件文件 (app.bin)
4. 点击上传
5. 等待设备重启

### App OTA

1. 通过 BLE 连接设备
2. App 发送 OTA 进入命令 (0x20)
3. App 传输固件数据
4. 设备自动校验并重启

## 常见问题

### Q: 编译报错 "esp-idf not found"

A: 确保已设置环境变量：
```bash
source ~/esp/v6.0.1/esp-idf/export.sh
```

### Q: 烧录失败 "Failed to connect"

A: 
1. 检查 USB 线是否支持数据传输
2. 按住 BOOT 按钮，再按 RST 进入下载模式
3. 尝试降低波特率：`--baud 115200`

### Q: 串口无输出

A:
1. 确认波特率设置为 1000000
2. 检查 TX/RX 引脚是否正确连接
3. 尝试按 RST 复位

### Q: TF 卡无法识别

A:
1. 确认 TF 卡格式化为 FAT32
2. 检查 SPI 引脚连接
3. 确认 Card_CD 引脚 (GPIO8) 连接正确

### Q: MAX30102 无数据

A:
1. 检查 I2C 引脚 (SDA=GPIO19, SCL=GPIO18)
2. 确认 MAX30102 模块供电正常
3. 检查手指放置是否正确

### Q: 电池电量显示不准

A:
1. 确认分压电阻值 (100K + 100K)
2. 检查 ADC 引脚 (GPIO4) 连接
3. 确认滤波电容 (100nF) 已焊接

### Q: BUTTON1 无反应

A:
1. 确认 BUTTON1 连接到 GPIO11
2. 检查是否有外部上拉电阻 (10k)
3. 确认按钮是低电平有效（接地 = 按下）
